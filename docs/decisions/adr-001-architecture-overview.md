# ADR-001: Core Architecture Decisions

**Status:** Accepted
**Date:** 2026-02-28

---

## Context

Mistral Raid is a 48-hour hackathon project. Architecture decisions prioritize demo reliability and "wow factor" over production hardening.

---

## Decisions

### WebSocket over REST

**Decision:** Use persistent WebSocket connection (not REST) for client↔server communication.
**Reason:** Eliminates per-request handshake latency. Critical when Mistral API already consumes 2–4s of the latency budget.

### JSON Mode over Function Calling

**Decision:** Use `response_format: { type: "json_object" }` rather than Mistral function calling.
**Reason:** Simpler to parse, fewer failure modes. No tool schema to maintain. JSON mode guarantees valid JSON even if the schema isn't perfectly followed.

### Lego Brick Pattern (No eval())

**Decision:** All mechanic types are pre-built classes. Mistral JSON maps to class constructors via a switch statement in `MechanicInterpreter.ts`. No code generation, no eval().
**Reason:** Security (no arbitrary code execution). Reliability (pre-tested classes). Type safety (each class has known parameters).

### Audio via REST + base64 (not WebSocket TTS)

**Decision:** ElevenLabs called via REST, audio returned as base64 string over existing WebSocket.
**Reason:** Simpler implementation for hackathon. WebSocket TTS streaming adds complexity for minimal gain (taunt is <30 words, <3s of audio).

### Single Railway Deployment

**Decision:** Backend Express server also serves `client/dist/` as static files.
**Reason:** One URL, no CORS configuration needed, simpler deployment.

### Fallback Cache Always Present

**Decision:** 5 hard-coded `BossResponse` objects in `client/src/config/fallbackAttacks.ts`.
**Reason:** API failures during a live demo would be catastrophic. Fallback must be instant and require zero network calls.

---

## Consequences

- MechanicInterpreter switch statement grows as mechanic types are added
- All 6 mechanic types must be implemented even if rarely generated by AI
- Single-server deployment means no horizontal scaling (acceptable for hackathon)
